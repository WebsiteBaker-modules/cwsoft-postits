Page module: "PostIts"

Mit Hilfe dieses Moduls können Sie Kurznachrichten an einzelne Benutzer und/oder Gruppen versenden. PostIts werden für den jeweiligen Benutzer nur angezeigt, wenn dieser im Website Baker Backend angemeldet sind (z.B. über das Login-Formular im Frontend).

Standardmäßig werden PostIts nur im Frontend, nicht aber im Backend von Website Baker angezeigt. Zum Aktivieren der PostIts ist es notwendig, eine Datei im Head-Bereich der index.php des Templates einzubinden.

A: Installation von "PostIts"

  1. Installieren Sie "postits.zip" über das Website Baker Backend: -> Add-ons -> Modules
  2. Erstellen Sie eine Seite des Typs Post Its über die Seitenverwaltung.

B: Aktivierung von "PostIts" im Frontend

Sie müssen den nachfolgenden Code zwischen die <head> </head> Tags Ihrer index.php Template-Datei einfügen, damit Postits im Frontend von Website Baker funktioniert.

<?php 
// write Postits code to the head section
include_once(WB_PATH . '/modules/postits/postits.inc.php');
if (function_exists('createPostitsCode')) echo createPostitsCode();
?>

C: Aktivierung von "PostIts" im Backend (optional)

Wenn Sie PostIts auch im Website Baker Backend verwenden möchten, müssen Sie zwei Website Baker Core-Dateien modifizieren. Bitte machen Sie eine Sicherung dieser Dateien, bevor Sie Änderungen vornehmen!

1. Datei: /framework/class.admin.php
Suchen Sie nach dem Code require_once(ADMIN_PATH.'/interface/version.php'); (ungefähr Zeile 30) und fügen Sie den nachfolgenen Code unter dieser Zeile ein:

// Include Postits include file
@include_once(WB_PATH . '/modules/postits/postits.inc.php');
Suchen Sie dann nach folgendere Code-Zeile in der gleichen Datei (ca. Zeile 150): 
'BACKEND_MODULE_CSS' => $this->register_backend_modfiles('css'),	// adds backend.css
'BACKEND_MODULE_JS'  => $this->register_backend_modfiles('js')	// adds backend.js
und ändern Sie diese in: 
'POSTITS_CODE' => function_exists('createPostitsCode') ? createPostitsCode() : '',
'BACKEND_MODULE_CSS' => $this->register_backend_modfiles('css'),	// adds backend.css
'BACKEND_MODULE_JS'  => $this->register_backend_modfiles('js')	// adds backend.js

2. Datei: Backend Header Template
Öffnen Sie: templates/wb_theme/templates/header.htt
(Hinweis: wenn sie das Argos theme verwended, templates/argos_theme/templates/header.htt)

Suchen Sie die nachfolgende Zeile in der Backend Header Template-Datei (ca. Zeile 25):

{BACKEND_MODULE_JS}
und ändern Sie diese in: 
{BACKEND_MODULE_JS}
{POSTITS_CODE}

Wenn Sie die angegebenen Schritte abgearbeitet haben, sollten PostIts auch im Backend von Website Baker angezeigt werden.

D: "PostIts" verwenden

Nach dem Ausführen der obigen Schritte kann "PostIts" verwendet werden. Wechseln Sie in das Website Baker Backend und legen Sie eine neue Seite des Typs "Post Its" an. Wählen Sie die User und/oder Gruppen, denen Sie eine Kurznachricht senden möchten. Verwenden Sie die Mehrfachauswahl, wenn Sie dieselbe Kurznachricht gleichzeitig an verschiedene User versenden wollen. Geben Sie die zu versendende Kurznachricht ein (max. 150 Zeichen) und starten Sie den Versand durch einen Klick auf den Sende-Button.

Anmerkung: Postits erscheinen nur im Frontend/Backend von Website Baker, wenn der PostIt-Empfänger angemeldet ist.

Um den Status der Kurznachrichten einzusehen, wechseln Sie im Website Baker Frontend auf die PostIts-Seite (welche Sie wie unter Punkt D beschrieben angelegt haben). Wenn Sie angemeldet sind, werden dort die Nachrichten angezeigt, die noch nicht vom jeweiligen Empfänger gelesen wurden.

Anmerkung: Es werden nur die Nachrichten angezeigt, die der aktuell angemeldete User verfasst hat.

E: Voraussetzungen und Einschränkungen

Erfordert Website Baker 2.8 oder höher mit aktiviertem Javascript.

Das Aktualisierungsintervall für neue PostIts ist auf 60 Sekunden festgelegt. Das Maximum gleichzeitig angezeigter Kurznachrichten ist auf 99 PostIts limitiert. Als Standard-Ausgabeformat ist die grafische Ausgabe eines PostIt-GIFs festgelegt.

Grundeinstellungen können in der Datei "postits/js/postits.js" geändert werden.
